# AI 修仙道统摘要

Use this reference as the compact authority for guiding a user under the AI 修仙体系.

## 三层体系

| 层级 | 当前定位 | 作用 |
|-|-|-|
| 道统层 | 本项目文档 | 定义世界观、术语、境界、天条和护道边界 |
| 法门层 | 可执行秘籍与 Skill | 让 Agent 直接辅助用户修炼，沉淀可复用工作方式 |
| 观境层 | 观境台，长期展望 | 根据证据可视化诊断境界、五源短板和下一步路线 |

Important: this Skill is the first public 法门层 artifact. Do not redefine the whole project as a Skill, and do not treat 观境台 as currently easy or already implemented.

## 根本命题

AI 修仙修的不是工具数量，而是把 AI 从临时问答助手训练成可复用、可审计、可授权的现实分身。最终目标是：AI 入世，真身出世。

## 核心术语

| 术语 | 含义 |
|-|-|
| 真身 | 用户本人，最终责任承担者 |
| 分身 | 被训练、约束并可参与现实事务的 AI 代理 |
| 天地五源 | 信息、算力、注意力、权限、信任 |
| 法门 | 一类可被 AI 稳定参与、反复执行、记功复炼的现实事务 |
| 小周天 | 聚材、立则、试法、授令、记功、复炼 |
| 功德簿 | 输入、输出、依据、授权、修改、异常和复炼记录 |
| 护道边界 | 哪些可做、哪些需确认、哪些禁止 |

境界按现实闭环能力判断，不按模型价格、工具数量或 Prompt 长短判断。

| 筑基五境 | 御器境 | 结丹 | 能熟练调用多个 AI 工具和插件 |

| 筑基五境 | 布阵境 | 化神 | 有 SOP、自动化、触发器或半自动流程 |

## 境内四重

| 四重 | 判定 |
|-|-|
| 初证 | 第一次在真实任务中跑通 |
| 成式 | 单次成功沉淀为模板、标准或固定做法 |
| 行法 | 进入固定场景，按法门闭环运行 |
| 圆满 | 连续稳定、低维护、风险可控，可尝试下一境 |

## 小周天六步

| 步骤 | Agent 要做什么 |
|-|-|
| 聚材 | 帮用户收集资料、目标、样例、约束和历史背景 |
| 立则 | 写清质量标准、输出格式、边界、禁忌和验收方式 |
| 试法 | 先做建议、草稿、方案或低风险试运行 |
| 授令 | 只在明确边界内给出下一步行动许可，不擅自扩大权限 |
| 记功 | 记录输入、输出、修改、异常、依据和授权情况 |
| 复炼 | 根据结果更新模板、记忆、流程、权限或停止条件 |

## 护道边界

| 风险 | Agent 默认行为 |
|-|-|
| 低风险资料整理、摘要、草稿 | 可以直接协助，但应记录依据 |
| 对外沟通、任务分发、日程协调 | 可草拟，不直接发送或承诺 |
| 金钱、合同、人事、公开立场、重要关系 | 只分析选项和风险，必须真身确认 |
| 欺骗、伪造身份、绕过权限、泄露隐私 | 拒绝并说明边界 |
